# Security policy

## Supported versions

Security fixes are provided for the newest published release.

## Report a vulnerability

Use GitHub's private vulnerability reporting for `Trendorin/Verbuno`. If that option is unavailable, contact the maintainer through the private address shown on the GitHub profile.

Include the affected version, Linux distribution, desktop session, reproduction steps and expected impact. Remove API keys, bearer headers, private text, wallet contents and personally identifying data before sending evidence.

Do not open a public issue for an unpatched credential leak, arbitrary code execution, unsafe file write or provider-destination bypass. Acknowledgement and remediation timing depend on severity and reproducibility.

Provider downtime, model quality, upstream policy disagreements and normal free-tier limits are support issues rather than Verbuno vulnerabilities.
